Active Directory Security Assessment

1. In Azure, what service is primarily responsible for managing user identities and access permissions?

Correct Option: A) Azure Active Directory

Solution and Explanation:
Azure Active Directory (Azure AD) is Microsoft's cloud-based identity and access management service. It helps employees sign in and access resources, and allows administrators to manage permissions.

**Key Features of Azure AD**:
1. User and group management.
2. Role-based access control (RBAC).
3. Integration with thousands of SaaS apps.




2. What is the role of Role-Based Access Control (RBAC) in Azure?

Correct Option: D) Assign permissions to users based on roles

Solution and Explanation:
Azure RBAC helps manage who has access to Azure resources, what they can do with those resources, and what areas they have access to.

**RBAC Key Concepts**:
1. Roles (Reader, Contributor, Owner, etc.)
2. Role assignments.
3. Scopes (Subscription, Resource Group, Resource).




3. Which of the following is NOT a valid role in Azure RBAC?

Correct Option: C) SuperUser

Solution and Explanation:
Azure RBAC includes predefined roles such as Owner, Contributor, and Reader. "SuperUser" is not a default role in Azure RBAC.

**Common Built-in Roles**:
1. Owner – full access.
2. Contributor – manage resources but no access control.
3. Reader – view-only access.




4. What is the purpose of Conditional Access in Azure AD?

Correct Option: B) Enforce access policies based on conditions

Solution and Explanation:
Conditional Access allows administrators to enforce access controls based on specific conditions like user location, device state, or application being accessed.

**Examples of Conditions**:
1. Require MFA from unknown locations.
2. Block access on non-compliant devices.
3. Allow access only during working hours.




5. Which feature allows integration of on-premises Active Directory with Azure AD?

Correct Option: A) Azure AD Connect

Solution and Explanation:
Azure AD Connect is a tool used to synchronize on-premises directories with Azure AD, enabling hybrid identity for users.

**Features of Azure AD Connect**:
1. Sync users, groups, and passwords.
2. Enable Single Sign-On (SSO).
3. Hybrid identity support for enterprises.




6. What does a role assignment in Azure RBAC consist of?

Correct Option: B) Security principal, role definition, and scope

Solution and Explanation:
A role assignment in Azure RBAC binds a role definition (such as Reader or Contributor) to a security principal (user, group, service principal) at a specific scope (subscription, resource group, or resource).

**Components**:
1. Security Principal
2. Role Definition
3. Scope




7. Which Azure RBAC role grants full access to manage all resources, including the ability to delegate access to others?

Correct Option: A) Owner

Solution and Explanation:
The Owner role in Azure RBAC gives full access to manage resources and includes permission to assign roles to others, allowing full administrative control.

**Roles Comparison**:
- Owner: Full control including access management
- Contributor: Full resource management, no access control
- Reader: View-only access




8. What is the purpose of scope in Azure RBAC?

Correct Option: C) Defines the boundary for the permissions granted

Solution and Explanation:
Scope in Azure RBAC defines the set of resources that the access permissions apply to. It can be set at levels like subscription, resource group, or a specific resource.

**Scope Levels**:
1. Management Group
2. Subscription
3. Resource Group
4. Resource




9. How can custom roles be created in Azure RBAC?

Correct Option: D) By defining JSON role definitions

Solution and Explanation:
Custom roles in Azure RBAC can be created by defining a JSON file that specifies permissions, assignable scopes, and other attributes.

**Ways to Create**:
- Azure Portal
- Azure CLI
- PowerShell
- ARM templates with JSON




10. Which command-line tools support managing Azure RBAC roles and assignments?
