41. What is the purpose of Soft-Delete in Azure Key Vault?

Correct Option: A) To allow recovery of deleted Key Vault objects within a retention period.

Solution and Explanation:
**Soft-Delete** is a feature that allows you to recover deleted Key Vault objects (like keys, secrets, and certificates) within a specific retention period.

**Key Features:**
1. Deleted items are retained for a configurable period (default 90 days).
2. Items can be recovered without data loss during this period.
3. After the retention period, items are permanently deleted unless Purge Protection is enabled.




42. What is Purge Protection in Azure Key Vault?

Correct Option: B) It prevents the permanent deletion of deleted Key Vault objects.

Solution and Explanation:
**Purge Protection** is a feature that prevents the permanent deletion (purge) of Key Vault objects even after the soft-delete retention period has expired.

**Key Features:**
1. Ensures that deleted items cannot be purged without explicit administrative action.
2. Provides an additional layer of protection against accidental or malicious deletions.
3. Must be enabled at the time of Key Vault creation or before an object is deleted.




43. How can you recover a deleted Key Vault object if Soft-Delete is enabled?

Correct Option: C) By using the Azure portal, PowerShell, or Azure CLI to restore the object.

Solution and Explanation:
If Soft-Delete is enabled, you can recover deleted Key Vault objects within the retention period using:

**Recovery Methods:**
1. **Azure Portal:** Navigate to Key Vault → Deleted Items → Restore.
2. **Azure CLI:** Use `az keyvault secret recover` or `az keyvault key recover` commands.
3. **PowerShell:** Use `Restore-AzKeyVaultSecret` or `Restore-AzKeyVaultKey` cmdlets.




44. What happens if Purge Protection is not enabled and the retention period expires?

Correct Option: A) The deleted Key Vault objects will be permanently deleted and cannot be recovered.

Solution and Explanation:
If Purge Protection is not enabled, deleted Key Vault objects will be permanently deleted after the retention period expires.

**Implications:**
1. Data cannot be recovered after permanent deletion.
2. Critical data loss risk if not managed properly.
3. It's recommended to enable Purge Protection for sensitive environments.
