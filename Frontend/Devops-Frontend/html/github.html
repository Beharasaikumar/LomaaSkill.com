GITHUB

1. What is the command to set a global Git username?

Solution and Explanation:
To set a global Git username, the correct command is git config --global user.name "username".




2. How can you check the current Git user email?

Solution and Explanation:
The correct command to check the current Git user email is git config user.email.




3. What command is used to clone a remote repository?

Solution and Explanation:
The correct command to clone a remote repository is git clone URL.




4. How do you initialize a new Git repository?

Solution and Explanation:
The correct command to initialize a new Git repository is git init.




5. What command adds all changes in the working directory to the staging area?

Solution and Explanation:
The correct command to add all changes to the staging area is git add .




6. What is the command to view the status of your working directory?

Solution and Explanation:
The correct command to view the status of your working directory is git status.




7. Which Git area is known as the "staging area"?

Solution and Explanation:
The correct answer is Staging Area (SA), which holds files before committing them.




8. How do you commit changes with a message?

Solution and Explanation:
The correct command to commit changes with a message is git commit -m "message".




9. Which command is used to push changes to a remote repository?

Solution and Explanation:
The correct command to push changes to a remote repository is git push origin.




10. How do you view the commit history for a specific file?

Solution and Explanation:
The correct command to view the commit history for a file is git log.




11. How do you stash changes with a message?

Solution and Explanation:
The correct command to stash changes with a message is A) "git stash -m 'message'".




12. Which command deletes a tag?

Solution and Explanation:
The correct command to delete a tag is C) "git tag -d".




13. What command is used to view all branches?

Solution and Explanation:
The command to view all branches in Git is "git branch". This command lists all local branches in your repository.
Therefore, the correct answer is C) "git branch".




14. Which command checks out a specific branch?

Solution and Explanation:
The command to switch to a specific branch is "git checkout" (or "git switch" in newer versions).
Therefore, the correct answer is B) "git checkout".




15. How do you merge two branches?

Solution and Explanation:
The command used to merge two branches in Git is "git merge".
Therefore, the correct answer is A) "git merge".




16. What is a merge conflict in Git?

Solution and Explanation:
A merge conflict occurs when two different changes are made to the same line in a file, and Git cannot automatically merge them. Developers must manually resolve these conflicts before completing the merge.
Therefore, the correct answer is C) A conflict that occurs when two different changes are made to the same line in a file.




17. How do you delete a local branch?

Solution and Explanation:
The correct command to delete a local branch is "git branch -d". If the branch has unmerged changes, use "git branch -D" instead.
Therefore, the correct answer is A) "git branch -d".




18. Why do we use the "git stash" command?

Solution and Explanation:
The "git stash" command is used to temporarily save changes in the working directory without committing them. This allows developers to switch branches or pull updates without losing their uncommitted work.
Therefore, the correct answer is A) To temporarily save changes in the working directory without committing.




19. What command allows you to amend the last commit?

Solution and Explanation:
The "git commit --amend" command allows you to modify the last commit by changing the commit message or adding new changes.
Therefore, the correct answer is B) "git commit --amend".




20. How can you apply changes from the stash?

Solution and Explanation:
The "git stash apply" command re-applies the last stashed changes without removing them from the stash. However, "git stash pop" also reapplies the changes but removes them from the stash.
Therefore, the correct answer is D) "git stash pop".




21. How do you view all stashes in Git?

Solution and Explanation:
The "git stash list" command displays all saved stashes.
Therefore, the correct answer is C) "git stash list".




22. Which command reverts a previous commit without modifying the working directory?

Solution and Explanation:
The "git revert" command creates a new commit that undoes changes from a previous commit without modifying the working directory.
Therefore, the correct answer is A) "git revert".




23. How do you delete a remote branch?

Solution and Explanation:
The command "git push origin --delete " is used to delete a branch from the remote repository.
Therefore, the correct answer is C) "git push origin --delete <branch>".




24. What command pushes all tags to the remote repository?

Solution and Explanation:
The "git push --tags" command pushes all local tags to the remote repository.
Therefore, the correct answer is B) "git push --tags".




25. How do you list all tags in a Git repository?

Solution and Explanation:
The "git tag" command lists all tags in the repository.
Therefore, the correct answer is D) "git tag".




26. Why do we use tags in GitHub?

Solution and Explanation:
Tags are used to mark specific points in the repository's history, often for releases.
Therefore, the correct answer is A) "To mark specific points in the repository's history, often for releases.".




27. Why do we use a .gitignore file?

Solution and Explanation:
The .gitignore file is used to specify files and directories that Git should not track.
Therefore, the correct answer is A) "To specify files and directories that Git should not track.".




28. How can you fork a repository on GitHub?

Solution and Explanation:
You can fork a repository by clicking the "Fork" button on the repository page.
Therefore, the correct answer is A) "By clicking the 'Fork' button on the repository page.".




29. What is a Personal Access Token (PAT) in Git?

Solution and Explanation:
A Personal Access Token (PAT) is a method of authenticating to GitHub that replaces passwords for API and Git operations.
Therefore, the correct answer is A) "A method of authenticating to GitHub that replaces passwords for API and Git operations.".




30. What is the main difference between CVS and DVCS?

Solution and Explanation:
CVS is centralized, while DVCS like Git is distributed.
Therefore, the correct answer is A) "CVS is centralized, while DVCS like Git is distributed.".




31. Which Git command would you use to display the commit history from the last 7 days?

Solution and Explanation:
The correct command is B) "git log --since='7 days ago'".




32. How do you move only file1 to the staging area from the working directory?

Solution and Explanation:
The correct command is A) "git add file1".




33. How can you restore a file from the staging area to the working directory?

Solution and Explanation:
The correct command is C) "git restore file2".




34. How can you check the logs of file9 and file45?

Solution and Explanation:
The correct command is "git log file9 file45", which shows the commit history for both files.




35. How can you check the logs for a specific user (e.g., user7)?

Solution and Explanation:
The correct command is "git log --author=user7", which filters commits by author.
