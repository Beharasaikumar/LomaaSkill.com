TERRAFORM

1. How is Terraform different from other IAC tools?

Solution and Explanation:
Terraform is declarative, focusing on defining the desired state, while others like Ansible are imperative.




2. What is the desired state and current state in Terraform?

Solution and Explanation:
Desired state is the state of infrastructure you want, and the current state is the real-world infrastructure after applying changes.




3. What is the state file in Terraform?

Solution and Explanation:
The state file stores the current state of your infrastructure.




4. What is "terraform init" and why do we need to do init?

Solution and Explanation:
"terraform init" initializes the working directory and downloads provider plugins.




5. What is "terraform plan", "validate", "fmt"?

Solution and Explanation:
"plan" checks potential changes, "validate" ensures correctness, "fmt" cleans up formatting, and "apply" makes changes.




6. How do you get all the resources created in the state file?

Solution and Explanation:
You can get all resources created in the state file by using the command terraform state list.




7. What are provisioners in Terraform and how do we use them?

Solution and Explanation:
Provisioners are used to run scripts or commands on a local or remote machine as part of resource provisioning.




8. How is the locking of a state file done in Terraform?

Solution and Explanation:
The state file is locked using backend services like S3 with DynamoDB for state file locking.




9. When should you use the "-target" option in Terraform?

Solution and Explanation:
The "-target" option is used when you want to target a specific resource for operations.




10. What does "terraform apply --auto-approve" do?

Solution and Explanation:
The command automatically approves and applies changes without a confirmation prompt.




11. What should you do when state files get corrupted?

Solution and Explanation:
You should use a recent backup of the state file or attempt to manually recover the state.




12. How do you restore a lost state file?

Solution and Explanation:
You restore the state file from backups or recover it from your version control system.




13. How do you use a data block in Terraform?

Solution and Explanation:
A data block is used to reference information or resources that exist outside Terraform.




14. Can you apply if-else logic in Terraform?

Solution and Explanation:
You can use conditional expressions with "count" or "for_each" in Terraform.




15. Suppose there is an active instance running, and you want to change its type from "t2.micro" to "t2.large". What should you do?

Solution and Explanation:
You need to modify the instance type in the configuration file and then run "terraform plan" followed by "terraform apply".




16. Which cloud providers does Terraform support?

Solution and Explanation:
Terraform supports a variety of cloud providers such as AWS, Azure, GCP, and others like DigitalOcean, Alibaba Cloud, and more.




17. In which language is Terraform code written?

Solution and Explanation:
Terraform code is written in HashiCorp Configuration Language (HCL), a custom language designed for this purpose.




18. What are workspaces in Terraform and which commands are related to them?

Solution and Explanation:
Workspaces allow managing multiple configurations in a single directory. Commands related to workspaces include "terraform workspace new", "terraform workspace select", and "terraform workspace list".




19. What is the major difference between Ansible and Terraform?

Solution and Explanation:
Terraform is declarative, focusing on desired state in infrastructure management, while Ansible is more of a configuration management tool that can use an imperative approach.




20. What are Terraform modules and how do we use them?

Solution and Explanation:
Modules are reusable groups of resources used to organize and structure complex configurations.




21. What is "surya", "upc1" and "upc2" in the following Terraform code?

Solution and Explanation:
"surya" is the AWS profile name used for authentication, stored in the ".aws" folder. "upc1" and "upc2" are module directories, where VPC configurations are stored.




22. A requirement to create two instances in two different regions using two different Terraform scripts at the same time. What is the best approach?

Solution and Explanation:
The best approach would be to use modules to manage each instance separately, applying the configurations at once.




23. How do you switch to a different workspace in Terraform?

Solution and Explanation:
You can switch workspaces using the command "terraform workspace select".




24. How can you list all available workspaces in Terraform?

Solution and Explanation:
You can list all available workspaces by using the command "terraform workspace list".




25. How do you delete a workspace in Terraform?

Solution and Explanation:
You can delete a workspace using the command "terraform workspace delete".




26. What happens to the state when switching between workspaces in Terraform?

Solution and Explanation:
Each workspace in Terraform has its own isolated state file, so when you switch between workspaces, you are switching to the state file associated with that workspace.




27. How do you view the current workspace in Terraform?

Solution and Explanation:
You can view the current workspace by using the "terraform workspace show" command.




28. Can you have different backend configurations for different workspaces?

Solution and Explanation:
No, all workspaces share the same backend configuration. You cannot define different backends for each workspace.




29. What is the default workspace in Terraform called?

Solution and Explanation:
The default workspace in Terraform is called "default" or "primary" depending on the version.




30. When is it a good practice to use multiple workspaces in Terraform?

Solution and Explanation:
It's a good practice to use multiple workspaces when managing multiple environments (e.g., dev, staging, prod) with the same configuration.




31. Why do we use variable Block and output Block in Terraform?

Solution and Explanation:
The "variable" block is used to define input variables, while the "output" block displays information after Terraform applies changes.




32. In your Terraform configuration, you want to provision a new EC2 instance and run a shell script to install Apache after the instance is created. Which provisioner would you use in your configuration?

Solution and Explanation:
You would use the "remote-exec" provisioner to run a shell script on the resource (EC2 instance) after it is created.




33. What is the primary difference between the local-exec, remote-exec, and file provisioners in Terraform?

Solution and Explanation:
The "local-exec" provisioner runs commands locally (on the machine running Terraform), the "remote-exec" provisioner runs commands on the resource after creation, and the "file" provisioner uploads files to the resource.




34. Find the error in the below script:

            resource "aws_instance" "instance1" {
                ami_id             = var.aws_ami_id
                key_pair           = "terraform_key"
                instance_type      = var.aws_instance_type
                availability_zone  = var.aws_AZ
                security_groups    = [aws_security_group.mysg.name]
                tags = {
                    Name = "instance-1"
                }
            }
Solution and Explanation:
The variable name should be corrected (e.g., `ami_id` instead of `ami_id`) and `key_pair` should be changed to `key_name`.




35. What does the below Backend.tf file explain?

            terraform {
                backend "s3" {
                    bucket         = "mys39178"
                    key            = "statefolder/terraform.tfstate"
                    region         = "us-east-1"
                    dynamodb_table = "terraform-lock-table"
                    encrypt        = true
                }
            }
Solution and Explanation:
The file configures S3 as the backend to store Terraform state files and uses DynamoDB for state locking.




36. What is the purpose of the terraform init command?

Solution and Explanation:
The "terraform init" command initializes the working directory containing Terraform configuration files. It also downloads the necessary provider plugins.




37. What does the terraform plan command do?

Solution and Explanation:
The "terraform plan" command shows you what changes will be made to the infrastructure, but does not apply them.




38. What is the purpose of the terraform validate command?

Solution and Explanation:
The "terraform validate" command checks for syntax errors in the Terraform configuration files and validates the configuration.




39. You have a Terraform-managed EC2 instance, and you suspect that the instance is not functioning correctly. You want to mark it for recreation without making any changes to your configuration. Which command should you use?

Solution and Explanation:
The "terraform taint" command is used to mark a resource for recreation. The resource will be destroyed and recreated during the next apply.




40. After marking an EC2 instance as tainted, you decide that it is functioning correctly and want to remove the taint mark. Which command would you use?

Solution and Explanation:
The "terraform untaint" command is used to remove the taint mark from a resource.




41. You taint a resource using the command "terraform taint aws_instance.my_instance". What will happen the next time you run "terraform apply"?

Solution and Explanation:
When you taint a resource, it is marked for recreation. The next time you run "terraform apply", Terraform will destroy and recreate that resource.




42. If you have multiple tainted resources, how can you see a list of all tainted resources in your Terraform state?

Solution and Explanation:
The command "terraform state list --tainted" will list all the tainted resources in your Terraform state.




43. What is Terraform Cloud primarily used for?

Solution and Explanation:
Terraform Cloud is primarily used to manage Terraform state files and provide a remote backend for Terraform runs.




44. Which feature in Terraform Cloud allows you to manage access controls and permissions for different users and teams?

Solution and Explanation:
Terraform Cloud allows you to manage access controls and permissions using "Teams", where you can define which users have access to certain workspaces and actions.




45. How does Terraform Cloud handle variable management?

Solution and Explanation:
Terraform Cloud allows you to define variables in the Cloud UI, enabling you to manage them separately for different workspaces.




46. Which of the following is a benefit of using Terraform Cloud over local Terraform runs?

Solution and Explanation:
Terraform Cloud provides a remote backend for state management, improving collaboration and security by ensuring that state files are centrally stored and securely accessed.




47. What is the purpose of the "Sentinel" feature in Terraform Cloud?

Solution and Explanation:
Sentinel is a policy as code framework used in Terraform Cloud to define and enforce rules that must be adhered to during Terraform runs. This helps in governance and compliance.




48. How can you trigger a run in Terraform Cloud?

Solution and Explanation:
You can trigger a run in Terraform Cloud by pushing changes to the main branch of the connected Git repository or manually clicking the "Queue Plan" button.




49. What is the name of the second instance in the following script?

        resource "aws_instance" "example" {
          count = 3
          ami = "ami-0c55b159cbfafe1f0"
          instance_type = "t2.micro"
          tags = {
            Name = "ExampleInstance-${count.index}"
          }
        }
Solution and Explanation:
The second instance will have the name "ExampleInstance-1" since the count starts at 0, so the second instance will be at index 1.




50. User1 has written a Terraform script and launched an EC2 instance. User1 has directly changed the instance type from t2.large to t3.medium in the AWS Cloud. When User1 runs the terraform refresh command, what happens?

Solution and Explanation:
The terraform refresh command will update the state file to reflect the current instance type in AWS (t3.medium) without changing the configuration. It does not revert the change made manually.
